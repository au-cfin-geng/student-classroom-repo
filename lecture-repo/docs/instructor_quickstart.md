# Instructor Quickstart

Concise reference for running, monitoring, and reviewing the Agentic Clinical Research Studio.

---

## Launch

```bash
# Start dashboard (local preview)
make dashboard

# Structural checks
make preflight    # 15 checks pass on fresh repo
make test         # full test suite

# Progress check
make progress     # always exits 0; shows all labs
```

Dashboard opens at http://localhost:8501.
Use `instructor-full.code-workspace` to see all repo internals.

## Grading modes

```bash
make progress        # friendly overview — always exits 0
make grade           # strict — exits 1 only if broken ready submission
```

Strict with required labs (for final grading):
```bash
python lecture-repo/scripts/grade_submissions.py \
  --mode strict \
  --required Lab00 Lab01 Lab02 Lab03 Lab04 Lab05
```

States in progress report:
- **Not started** — submission folder has only README
- **In progress** — package run but not complete
- **Ready** — `ready_for_review=true`, all evidence present, full submission.md
- **Broken** — claims ready but evidence or structure missing

---

## What GitHub Actions checks

On every push touching `Lab*/` or `*Submission/`:
1. `make progress` — always exits 0; written to grading report artifact
2. `make grade` (strict mode) — exits 1 only if a broken ready submission is found

Download grading report: Actions tab → run → Artifacts → `grading-report-<sha>.zip`

## Submission review priority

Focus manual review on these labs in order:

1. **Lab01 — Prompt Contract** (`Lab01Submission/submission.md`)
   Look for: output contract specificity, permission design, role specification quality

2. **Lab03 — Literature Search Skill** (`Lab03Submission/submission.md`)
   Look for: skill reusability, search methodology, claim grounding

3. **Lab05 — Subagent Workflow** (`Lab05Submission/submission.md`)
   Look for: decomposition reasoning, handoff design, orchestration honesty

4. **Capstone** (`CapstoneSubmission/submission.md`)
   Look for: integration of multiple labs, independent research design, showcase summary

## Reviewing a submission

```bash
# Check overall progress
make progress

# Read a specific submission
cat Lab03Submission/submission.md

# Check evidence exists
ls Lab03/work/

# Read the evidence manifest
cat Lab03Submission/evidence_manifest.json
```

Key questions for manual review:
- Does submission.md name specific artifacts from `LabXX/work/`?
- Are limitations honestly reported?
- Does the student describe what they actually verified vs. accepted?
- Is the clinical reasoning grounded in evidence?
