Agentic Clinical
Research Studio

A two-day clinical research lab where medical and life-science PhD students learn to use Claude, VS Code, and GitHub to build traceable, reviewable agentic research workflows.

Start here before opening the repository

Curriculum

What you will learn

Each lab teaches one research principle through a concrete clinical scenario. All work is done through structured prompts — no prior programming required.

Lab 01

Prompt Contracts

Turn vague clinical research requests into verifiable, auditable tasks with explicit output criteria and success conditions.

Lab 02

Stakeholder Review

Use Claude to review a study from IT, clinician, legal, PI, reviewer, and public perspectives simultaneously.

Lab 03

Literature Search Skills

Build reusable Claude workflows for translating clinical questions into rigorous, reproducible search strategies.

Lab 04

Tool / MCP-Aware Workflows

Design safe, auditable ways for Claude to connect to files, papers, datasets, and external research tools.

Lab 05

Subagent Workflows

Design a small agentic research team for dividing, reviewing, and synthesising clinical research tasks.

Capstone Mini-Project Apply the complete workflow to your own clinical research question.

Course model

How the course works

Every lab follows the same loop. Claude is your research partner — not an answer machine. You write the prompt; Claude produces the artifact; you evaluate it.

Read the lesson Open the dashboard with make dashboard and read the lab brief in the sidebar.
Copy the lab prompt into Claude Code Each lab has a ready-made prompt. Paste it into Claude Code inside VS Code.
Claude works inside LabXX/work/ Claude produces structured research artifacts — documents, plans, search strategies, reviews.
You inspect the artifact Read Claude's output critically. Is it grounded? Does it meet the research standard? Is it honest about uncertainty?
Claude packages the submission Paste the Finish Lab prompt — Claude creates LabXXSubmission/ automatically. You do not run commands.
GitHub checks progress GitHub Actions verifies the submission structure on every push. Your instructor reviews selected submissions.

The student mental model

You work in LabXX/work/.

Claude packages LabXXSubmission/ using the Finish Lab prompt.

GitHub Actions checks structure automatically on every push.

The instructor reviews selected submissions.

Setup

Setup checklist

Complete these six steps once, before opening the repository. After setup, everything runs through Claude Code and the course dashboard.

1

Install VS Code

Download and install Visual Studio Code for your operating system.

code.visualstudio.com/Download →
2

Install Claude Code

Claude Code lets Claude read the project, edit files, run commands, and help prepare submissions. Follow the official installation guide.

Claude Code overview and installation →
3

Add Claude Code to VS Code

Open the Extensions panel (Cmd+Shift+X on Mac · Ctrl+Shift+X on Windows), search for Claude Code, and install. Sign in when prompted.

Claude Code VS Code integration →
4

Create a local course folder

Create a new, empty folder on your computer for the course. Keep it separate from other projects.

Example: Documents/agentic-clinical-research-course/

5

Open that folder in VS Code

File → Open Folder → choose your course folder. The Explorer panel should show an empty directory.

6

Open Claude Code and paste the setup prompt

Open Claude Code inside VS Code. Copy the prompt below and paste it into the Claude Code chat. Claude will clone the repo, run checks, and walk you through the next steps.

Setup prompt

Copy and paste this into Claude Code

This prompt tells Claude to clone the course repo, run setup checks, and explain how to start. Paste it once, into the Claude Code chat inside VS Code.

Claude Code — paste into chat 
You are Claude Code helping me set up the Agentic Clinical Research Studio course repo.

I am a student. I want you to prepare the course workspace for me.

---

FIRST — confirm where you are:

* Tell me the current folder path.
* Tell me whether it is empty or already contains the course repo.
* If the repo is already here, skip the clone step.
* Do not delete or overwrite anything without asking me first.

---

SECOND — get the repository URL:

* If my instructor gave me a GitHub Classroom repository URL,
  ask me to paste it now and use that URL.
* If I do not have a Classroom URL, use the instructor template repo:
  git@github.com:au-cfin-geng/student-classroom-repo.git
* If SSH authentication fails, stop immediately and tell me
  what to report to the instructor.

---

THIRD — clone and inspect:

* Clone the repository into this folder.
* Enter the cloned folder.
* Read START_HERE.md and CLAUDE.md so you understand the course structure.
* Tell me in 3 sentences what this course is about.

---

FOURTH — run setup checks:

* Run: make preflight
  Tell me whether it passed or failed.
  If it failed, explain what is missing.

* Run: make progress
  Show me the current lab progress table.

---

FIFTH — explain how to work:

Tell me:

1. Open student-lab.code-workspace in VS Code
   to get the full course view.
2. Run `make dashboard` to open the course dashboard
   (the visual interface for all labs).
3. All lab work goes into LabXX/work/
   — for example, Lab01/work/.
4. When I finish a lab activity, I paste the Finish Lab prompt
   from lecture-repo/prompts/finish_lab.md into Claude Code.
5. Claude packages the submission into LabXXSubmission/
   — I do not run any commands manually.
6. I can check progress anytime with `make progress`.

---

RULES:

* Do not push anything during setup.
* Do not modify lecture-repo/ unless running course setup commands.
* Do not create or edit student submission files during setup.
* Do not make assumptions about my GitHub account or authentication.
* If any step fails, stop and explain clearly what I should
  tell the instructor.

If your instructor gave you a GitHub Classroom repository URL, use that URL instead of the template repo when Claude asks.

After setup

After Claude finishes setup

Once Claude confirms preflight passes and shows you the progress table, your workspace is ready. Follow these steps to start the course.

Open student-lab.code-workspace in VS Code for the full course view.
Run make dashboard if Claude has not already started it, and open the URL it prints.
Start with Lab00 — read the brief in the dashboard, then paste the lab prompt into Claude Code.
Work only inside LabXX/work/. Do not manually create or edit submission folders.
Finish each lab using the Finish Lab prompt from the dashboard or from lecture-repo/prompts/finish_lab.md.
Check progress anytime with make progress or in the dashboard.

Help

If you get stuck

Setup problems are common and fixable. Tell your instructor or TA the following, and most issues resolve quickly.

What to report

  • Which setup step failed — or which lab you were on
  • What Claude said — copy the exact message
  • What error appeared — copy the full error text
  • Whether the repo was cloned successfully
  • Whether make preflight passed or failed, and what it printed
