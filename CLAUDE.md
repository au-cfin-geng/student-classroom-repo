# CLAUDE.md — Lab Instructions for Claude Code

Read this file before taking any action in this repository.
This file is the project-level contract between you and the student.

---

## Course identity

This is the **Agentic Clinical Research Studio** — a lab-based curriculum teaching clinical and life-science researchers to design agentic research workflows that produce traceable, reviewable, clinically responsible artifacts.

The tools taught: Claude, VS Code, GitHub, Skills, tools/MCP, structured prompts, stakeholder roles, and subagents.

The student is a **junior clinical AI investigator**. Their job is to write precise prompts that function as research protocols, judge Claude's outputs against real research standards, and commit artifacts that constitute an auditable research record.

## The golden rule

**All student-generated files for a lab go in `LabXX/work/`.**

Do not write student outputs anywhere else. Do not write into `lecture-repo/`. Do not scatter files in the repo root.

---

## Lab interaction model

1. Student opens dashboard: `make dashboard`
2. Student reads the lab in the dashboard sidebar
3. Student pastes `LabXX/do_lab_prompt.md` into Claude Code
4. Claude produces artifacts in `LabXX/work/`
5. Student runs `make package LAB=LabXX` → creates `LabXXSubmission/`
6. Student fills in `LabXXSubmission/submission.md`
7. Student commits and pushes — GitHub Actions verifies structure

## Design authority

**Primary design specification:** `lecture-repo/docs/` directory

If any instruction here conflicts with the spec documents, the spec takes precedence.

## Minimal-edit philosophy

- Read the lab prompt in `LabXX/do_lab_prompt.md` before making any changes
- State your plan in plain language before editing any file — the student should approve it
- Edit the smallest number of files needed to advance the current lab
- Focus on one lab at a time; do not race ahead
- After completing a lab, verify that expected artifacts exist in `LabXX/work/`

---

## Reproducibility requirements

- Scripts must run from the repo root
- Use `Path(__file__).resolve().parents[2]` in `lecture-repo/scripts/` for the base path
- Do not hardcode absolute paths
- Scripts must be idempotent — safe to run more than once

---

## Honesty requirements

- Never fabricate or invent metric values
- If a result is poor, report it honestly and explain what the numbers mean
- If a command raises an error, diagnose and fix the root cause — do not suppress errors
- If a lab cannot complete due to missing data, say so clearly

## Research language requirement

Explain all actions in research language, not software engineering language.

Say "I am writing the prompt contract to `Lab01/work/L1_prompt_contract.md`" not "I am saving the file."

Say "This orientation artifact records your workspace readiness" not "I am generating the JSON."

This is a clinical research lab, not a coding bootcamp.
